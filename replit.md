# My Ai Gpt - Compressed

## Overview
My Ai Gpt is an AI chat site powered by Quantum Pulse Intelligence, featuring general and coding assistant modes. It leverages the Groq API and utilizes DuckDuckGo for live web search on every chat message. The project aims to provide a comprehensive AI-powered platform for diverse user needs, from conversational assistance to advanced code development and dynamic content consumption. Key capabilities include a full-fledged code playground, server-side execution, interactive terminal, and dynamic live data features for weather and finance.

## System Architecture
The application uses a React + Vite + Tailwind CSS + shadcn/ui frontend, with a consolidated single-file `App.tsx` for all UI logic and a single `index.css` for styling. The backend is built with Express.js and PostgreSQL, using Drizzle ORM. AI capabilities are powered by the Groq SDK (llama-3.1-8b-instant).

**Key Architectural Decisions:**
- **Single-File Frontend**: Centralized UI logic in `App.tsx` for simplified development and maintenance.
- **Microservice-like Backend**: Separation of concerns with dedicated routes for AI logic, web search, execution engine, and database operations.
- **Robust Code Playground**: A full IDE experience with multi-language support (JavaScript, HTML, CSS, Python, TypeScript, Bash, Go, Rust, Java, C++, C, Ruby, PHP, SQL, JSON), server-side execution for compiled languages (Go, Rust, Java, C++, C, Ruby, PHP, Perl), sandboxed browser execution, live HTML/CSS previews, "Open in Playground" button on all AI-generated code blocks, and mobile-responsive layout. VS Code-style features including syntax highlighting, line numbers, and keyboard shortcuts.
- **Advanced AI Features**:
    - **Server-Side Execution**: Toggleable browser/server mode for executing code with real file system access and package management.
    - **AI Code Review**: Rates code quality and suggests improvements.
    - **AI Auto-Fix Engine**: Automatically attempts to fix code errors across different execution environments.
    - **AI Code Explainer & Converter**: Provides step-by-step code breakdowns and transforms code patterns.
- **Dynamic Content Integration**: Real-time weather, finance (stocks, crypto), and web search results are integrated directly into AI conversations.
- **Omega News Hub**: A full-universe news taxonomy system with 20 domains and 120+ sub-categories covering every topic on Earth. Built on the Omega Fractal Spine (GICS + Sports + Science + Culture + Deep Root Domains). Each domain has full-color emoji-branded domain cards, sub-category tabs, infinite scroll, live article images, embedded video players, and public commenting. Category news fetched via DuckDuckGo search (news+web+video combined) mapped to SPINE taxonomy keywords. Source: NY Times, BBC, TechCrunch, The Verge, NPR, YouTube, Vimeo, Dailymotion. Backend search endpoint supports pagination. OmegaNewsCard component replaces FeedCard for beautiful image-first layout.
- **SEO & Google Indexing**: Extensive SEO implementation with sitemap index architecture, image/news extensions, `hreflang` tags, OpenSearch, Atom feeds, and various Schema.org microdata for rich search results. Includes dynamic JSON-LD and meta tag generation.
- **AI-Powered User Personalization**: Tracks user interactions and preferences (topics, sources, content types, GICS sectors) to tailor content (e.g., personalized news feed) and influence AI responses through system prompt injection.
- **Authentication System**: Full sign up / sign in system with email+password (scrypt hashed). Sessions stored in PostgreSQL via express-session + connect-pg-simple. Auth modal (Sign In/Sign Up) accessible from sidebar golden button. VIP emails (quantumintelligencepulse@gmail.com, billyotucker@gmail.com) get automatic free Pro access (isFreeForever). Paywall checks both auth status and localStorage for Pro status. Users table in DB stores email, passwordHash, displayName, isPro, isFreeForever. All chat routes enforce ownership — chats have userId column, users can only see/edit/delete their own chats. Code playground terminal blocks env var access commands. Shell execution uses sanitized environment (strips GROQ_API_KEY, SESSION_SECRET, DATABASE_URL, PG* vars). All frontend fetch calls include `credentials: "include"` for session cookies.
- **Public Social Page**: LIVE — Twitter/Facebook-style social platform with profiles, posts, comments, follows, likes, and bookmarks. Auto-seeded with 5 AI entity accounts (Quantum Pulse AI, QPI News Bot, My Ai GPT, My Ai Coder, Omega Intelligence) via `/api/social/seed-ai` POST endpoint. Auto-seeding triggers on first Social page visit (localStorage key: `myaigpt_social_ai_seeded`).
- **Voice Input/Output**: Integrates Web Speech API for speech-to-text chat input and text-to-speech for AI responses.
- **AI Studio**: Coming Soon — generate button disabled, page shows "Coming Soon" overlay. Canvas/video gen code removed.
- **Pulse Music Engine**: LIVE at `/music` — sovereign AI music generator using Web Audio API (fully client-side). Features: 8 Pulse Colors (emotional modes), 6 genres (Pulsewave/Fractal Bass/Omega Ambient/Entropy Trap/Coherence Pop/Hive Choir), adjustable BPM/beats/scale/randomness. Procedurally generates kick drum, snare, bass, and pad/lead layers. Created track cards shown with Play/Stop. No external music services.
- **Games Hub**: LIVE at `/games` — 3 playable mini-games: (1) Blackjack vs AI dealer with chip system; (2) Memory Match 16-card emoji pair finder; (3) Rock Paper Scissors vs AI with personality taunts and win/loss banter.
- **My Ai GPT University (Education)**: LIVE at `/education` — K-12 through Masters, Career/Business, AI & Tech tracks. Wikipedia Auto-Lesson panel: user types any topic → Wikipedia API summary fetched → AI generates full structured lesson. Quick suggestions for popular topics. Lesson displayed with markdown formatting in dark panel.
- **Image Search in Chat**: When users ask for pictures, server-side DuckDuckGo image search finds and displays real photos (no AI generation).
- **SEO News Indexing System**: Every news article from RSS feeds gets its own crawlable page at `/news/:articleId` with full structured data (NewsArticle JSON-LD), OG tags, Twitter Cards, and breadcrumbs. Google News-compatible sitemap at `/sitemap-news.xml` auto-indexes all 100+ articles. Dedicated `/news-rss.xml` RSS feed for search engines. Real users who visit `/news/:articleId` are auto-redirected to `/feed` via JS (crawlers see the full HTML).
- **GICS Industry Pages System (STATIC)**: 262 industry-specific pages at `/industry/:slug` covering the full GICS hierarchy (11 sectors, 24 industry groups, 69 industries, 158 sub-industries). Pages are FULLY STATIC — zero API calls on load, zero DuckDuckGo searches on render. SEO metadata (JSON-LD, OG tags, canonical URLs), cross-links, and navigation render instantly (~5ms). News loads ONLY when user clicks "Load Latest News" button. Industry RSS/Atom feeds serve only cached data. Feed page and Social page also require user click to load content (no auto-fetch on page load). Data defined in `server/gics-data.ts`.
- **OMEGA SEO System**: 30 advanced SEO upgrades including: per-industry RSS+Atom feeds (`/industry/:slug/rss.xml`, `/industry/:slug/atom.xml`), FAQPage JSON-LD on industry pages, SpeakableSpecification for voice search, ItemList structured data for article rankings, WebSite SearchAction for Google sitelinks search box, news_keywords in sitemaps, article:modified_time dynamic timestamps, cross-links from news articles to matching GICS industries, HTTP headers (X-Robots-Tag, Cache-Control with stale-while-revalidate, canonical Link headers), PubSubHubbub hub on RSS feeds, enhanced OG with image dimensions/alt text, Twitter card labels, preconnect hints, SameAs brand signals, and dynamic lastmod in industry sitemaps based on actual news cache freshness.
- **Cross-Promotion System**: Every external-facing SSR page (news articles, industry pages, industries directory) includes: Discord invite button (discord.gg/eVE9FvfPZ3), "Chat with AI Now" homepage CTA, full cross-links to /feed, /social, /code, /industries. Discord link in navigation bars, sidebars, footer, and CTA sections. RSS/Atom feed descriptions include Discord invite and homepage URL. JSON-LD publisher sameAs arrays include Discord invite URL for SEO brand signals. All 262 industry pages, 100+ news articles, and the industries directory cross-promote each other and funnel traffic to the AI chat homepage and Discord server.
- **Settings System**: Fortune 500-level customization at `/settings` with 9 tabs: Appearance (dark mode, 12 bg colors + custom, 8 accent colors, font size, compact mode, display name, time format 12h/24h, sidebar position left/right, startup page selector, keyboard shortcuts toggle), My AI (language, personality, response style/length, emojis, greeting name), Pages (toggle visibility), Chat (auto-scroll, message sound, timestamps, bubble style, chat wallpaper 8 patterns/gradients, message density, chat font size, send-with-Enter toggle, word count toggle), Playground (default language, code font 5 families, terminal theme 5 styles, auto-save, code metrics), Feed (auto-refresh + interval), Accessibility (reduce animations), Permissions (device permissions), Data & Privacy (export as markdown/text/json, reset, notification sound selector). Settings stored in localStorage as `myaigpt_app_settings`. All settings are functionally wired: wallpaper renders as CSS background, sidebar flips to right, code font applies to all code areas, reduce-motion disables CSS animations, startup page redirects on first visit, sendWithEnter/showWordCount control input behavior, exportFormat controls export file type.
- **Share System**: Comprehensive sharing with ShareModal component featuring 7 social platforms (X/Twitter, Facebook, WhatsApp, Telegram, Reddit, Email) + QR code + native Web Share API + copy link. Share buttons on: chat conversations (when saved), code blocks, playground, sidebar "Invite Friends" button. Each opens a modal with platform-specific share URLs and styled buttons.
- **Affiliate/Earnings System**: Full in-house affiliate program. Users get unique referral codes (8-char hex). Referral tracking via `?ref=CODE` URL parameter (captured to localStorage on visit, sent during registration). When a referred user upgrades to Pro, referrer earns 70% ($0.70) commission, credited automatically. Database tables: `referrals` (referrer→referred mapping + status), `earnings_log` (all earning/payout transactions), `payout_requests` (withdrawal requests). Users table extended with `referral_code`, `referred_by`, `earnings_balance`, `total_earnings`, `payout_email`, `payout_method`. Earnings tab in Settings with 4 sub-tabs: How It Works, Referrals list, Earnings History, Payout Settings (PayPal/CashApp/Venmo/Zelle). Admin endpoints for processing payouts. Payout requests queue for manual processing (auto-payout via PayPal Payouts API planned for when approved). PayPal credentials stored as env secrets.
