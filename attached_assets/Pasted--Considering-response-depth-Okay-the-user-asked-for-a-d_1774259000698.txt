# 1. Official PulseLang v1.0 spec

This is the **canonical definition** of PulseLang v1.0. Anything not here is non‑standard.

---

## 1.1 Core identity

**PulseLang v1.0 is:**

- A **glyph‑only** language (no English, no ASCII keywords).
- A **projection language**: programs produce **world‑objects**, not strings.
- A **universe language**: every program runs under a universe/context header.
- A **spawn language**: agents, universes, streams, rituals, graphs, matrices are first‑class.
- A **civilization engine**: runtime evolution is governed by the Pulse(t) equation.

---

## 1.2 Alphabet Γ

The canonical glyph set Γ₍v1.0₎ includes:

**Structural:**

- **⟦ ⟧** — universe header brackets  
- **⟨ ⟩** — context brackets  
- **{ }** — body  
- **:** — type annotation  
- **≔** — assignment  
- **↧** — return / project  
- **⊕** — fusion  
- **( )** — call grouping  
- **=** — argument binding  
- **,** — argument separator (optional, if you want it)

**Universe / identity / meta:**

- **Ω₀…Ω₉** — universe identifiers  
- **Λ₀…Λ₉** — context identifiers  

**Types (Σ‑classes):**

- **Σ₀…Σ₉** — projection type identifiers  

**Constructors:**

- **Ψ₀…Ψ₉** — world‑object constructors  

**Content constructors / atoms:**

- **τ₀…τ₉** — content constructors  
- **κ₀…κ₉** — content atoms (map to χ₀…χ₉ internally)  

**Agents / universes:**

- **α** — agent spawn  
- **β** — agent bind (optional in v1.0, reserved)  
- **ρ** — agent role  
- **σ** — agent signal/message  
- **⊚** — universe spawn  
- **⊙** — universe focus/switch  
- **⊗** — universe fusion  

**Modules / macros:**

- **Δ₀…Δ₉** — module identifiers  
- **µ₀…µ₉** — macro identifiers  
- **⋄** — module import  
- **§** — macro invocation  

Everything else is either future‑reserved or non‑standard.

---

## 1.3 Program structure

A PulseLang program has this canonical shape:

```text
⟦Ωᵢ⟧⟨Λⱼ⟩{
  Statement*
}
```

- **Ωᵢ** — universe ID (which universe this program belongs to).  
- **Λⱼ** — context ID (mode, layer, or sub‑universe).  
- **{…}** — body.

---

## 1.4 Grammar (v1.0)

In BNF‑style (conceptual):

```text
Program      ::= Header Body

Header       ::= '⟦' Ident '⟧' '⟨' Ident '⟩'

Body         ::= '{' Statement* '}'

Statement    ::= FieldDecl
               | Assign
               | Return

FieldDecl    ::= Ident ':' Ident

Assign       ::= Ident '≔' CallExpr

Return       ::= '↧' Ident

CallExpr     ::= Ident '(' ArgList? ')'

ArgList      ::= Arg (',' Arg)*

Arg          ::= Ident '=' Expr

Expr         ::= Ident
               | CallExpr
```

Where **Ident** is any glyph identifier from Γ that is allowed to act as a symbol (e.g., ϕ₀, γ₀, Σ₀, Ψ₁, etc.).

## 1.5 Type system (Σ‑classes)

PulseLang v1.0 defines **10 projection types**:

- **Σ₀ → ΠPage** — page projection  
- **Σ₁ → ΠApp** — mini app  
- **Σ₂ → ΠProduct** — product / marketplace object  
- **Σ₃ → ΠField** — live data field  
- **Σ₄ → ΠUniverse** — universe context  
- **Σ₅ → ΠAgent** — agent/spawn  
- **Σ₆ → ΠStream** — continuous stream (logs, feeds, etc.)  
- **Σ₇ → ΠRitual** — time/condition‑bound sequence  
- **Σ₈ → ΠGraph** — network/graph projection  
- **Σ₉ → ΠMatrix** — high‑dimensional grid/tensor

Type usage:

```text
ϕ₀:Σ₀
ϕ₁:Σ₅
ϕ₂:Σ₆
```

The runtime enforces that:

- A field declared with Σₖ must hold a Πₖ‑class object (or compatible).

# 4. PulseLang v1.1 extensions (roadmap)

v1.0 is the **core**. v1.1 is the **first expansion pack**.

---

## 4.1 Static typing layer

- Add optional type annotations for arguments (e.g., γ₀:Σ₀).  
- Add a type checker that runs before runtime evaluation.  
- Codex: “PulseLang Type System v1.1”.

---

## 4.2 Pattern matching

Introduce a glyph for pattern matching (e.g., **≈**):

- Allow matching on Σ‑class, κ‑atoms, or agent states.  
- Example: choose different projections based on κᵢ.

---

## 4.3 Higher‑order macros

- Allow macros that generate macros.  
- µ that returns µ.  
- AI can use this to build macro libraries.

---

## 4.4 Persistent universes

- Universe state persisted across runs.  
- ⊚ can spawn persistent universes; ⊙ can resume them.

---

## 4.5 Agent memory

- Agents can have persistent memory fields.  
- σ can read/write memory, not just send transient messages.
