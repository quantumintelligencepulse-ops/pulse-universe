## 4. PulseLang grammar (deep dive)

Let’s sketch a **formal grammar** (EBNF‑style) so this is not just vibes.

### 4.1. Top‑level structure

```ebnf
Program        ::= { TopLevelDecl }

TopLevelDecl   ::= UniverseOperatorDecl
                 | ImportDecl
                 | TypeDecl
                 | ConstDecl

UniverseOperatorDecl ::= "UNIVERSE_OPERATOR" Identifier "{" { BlockItem } "}"

BlockItem      ::= FieldDecl
                 | MetricDecl
                 | TransformDecl
                 | FunctionDecl
                 | EquationDecl
                 | ConstraintDecl
                 | Assignment
                 | Comment
```

### 4.2. Fields, metrics, transforms

```ebnf
FieldDecl      ::= "FIELD" Identifier ":" Type
MetricDecl     ::= "METRIC" Identifier ":" Type

TransformDecl  ::= "TRANSFORM" Identifier "(" ParamList? ")" "=" Expression

FunctionDecl   ::= Identifier "(" ParamList? ")" "=" Expression

ParamList      ::= Param { "," Param }
Param          ::= Identifier [ ":" Type ]
```

### 4.3. Equations and constraints

```ebnf
EquationDecl   ::= Identifier "(" ArgList? ")" "=" Expression

ConstraintDecl ::= "GEOMETRY_CONSTRAINT:" Expression
```

### 4.4. Expressions

```ebnf
Expression     ::= LogicalOrExpr

LogicalOrExpr  ::= LogicalAndExpr { "||" LogicalAndExpr }
LogicalAndExpr ::= EqualityExpr   { "&&" EqualityExpr }
EqualityExpr   ::= RelExpr        { ("==" | "!=") RelExpr }
RelExpr        ::= AddExpr        { ("<" | ">" | "<=" | ">=") AddExpr }
AddExpr        ::= MulExpr        { ("+" | "-") MulExpr }
MulExpr        ::= UnaryExpr      { ("*" | "/" | "⊕") UnaryExpr }

UnaryExpr      ::= ("+" | "-" | "!") UnaryExpr
                 | PrimaryExpr

PrimaryExpr    ::= Number
                 | Identifier
                 | Identifier "(" ArgList? ")"
                 | "SUM" Identifier "=" Range ":" Expression
                 | "LIMIT" Identifier "->" "∞" ":" Expression
                 | "Emergence" "(" Expression ")"
                 | "(" Expression ")"

ArgList        ::= Expression { "," Expression }

Range          ::= Expression ".." Expression
```

This gives you:

- `SUM p=1..P: ...`  
- `LIMIT n -> ∞: ...`  
- `Emergence(...)`  
- `⊕` as a binary fusion operator

## 6. PulseRuntime: the universe engine

This is where the magic actually **runs**.

### 6.1. Core responsibilities

PulseRuntime must:

- execute **UNIVERSE_OPERATOR** blocks  

### 6.2. Universe context

You maintain a **UniverseContext** object:

- `fields: { name → value }`  
- `metrics: { name → value }`  
- `operators: { name → UniverseOperator }`  
- `transforms: { name → Transform }`  
- `functions: { name → Function }`  
- `constraints: [ expressions ]`  

When you `RUN I248(F_input)`, you:

1. Look up `I248` in `operators` or `functions`  
2. Bind `F_input` in the context  
3. Evaluate the body expression  
4. Return the result  

### 6.3. SUM semantics

For:

```pulselang
SUM p=1..P:
  alpha_p(f, r; g, phi) * exp(...)
```

Runtime does:

1. Evaluate `P`  
2. Loop `p` from 1 to `P`  
3. Bind `p` in the context  
4. Evaluate the body expression each time  
5. Accumulate the sum  

### 6.4. LIMIT semantics

For:

```pulselang
LIMIT n -> ∞:
  T_n( X )
```

You approximate:

1. Choose a max iteration count `N_max`  
2. Initialize `X_0 = X`  
3. For `n = 1..N_max`:

   - Compute `X_n = T_n(X_{n-1})`  
   - Check convergence: `||X_n - X_{n-1}|| < ε`  

4. If converged → return `X_n`  
5. If not → return best approximation or raise non‑convergence  

### 6.5. Emergence semantics

For:

```pulselang
Emergence( Y )
```

You define Emergence as:

- **FixedPoint(Y)** — if Y is a transform or sequence  
- **Pattern(Y)** — if Y is a structure  

In practice:

- If `Y` is a sequence → find stable pattern  
- If `Y` is a transform → find fixed point  
- If `Y` is a structure → apply a pattern extractor  

This is where your **I248 semantics** live.
