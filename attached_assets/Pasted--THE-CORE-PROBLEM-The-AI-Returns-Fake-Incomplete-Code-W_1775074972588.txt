🔴 PROBLEM 1: The Prompt Isn't Strict Enough
The current prompt asks for html, css, js as separate JSON fields. Claude then writes them as if they're separate files, but:

The JS does document.getElementById("btn") but the HTML never has id="btn"
The CSS has .dark-mode but JS never toggles it
External libraries are imported but the iframe has no internet access to CDNs
THE FIX: The prompt must demand ONE complete self-contained HTML string — not split files. Everything inline. Like this:

Return ONE complete HTML file as a single string.
ALL CSS in a <style> tag inside <head>.
ALL JavaScript in a <script> tag at the end of <body>.
NO external imports. NO CDN links. NO fetch() calls.
The file must work by opening it in a browser with zero internet.
🔴 PROBLEM 2: The iframe SrcDoc Is Assembled Wrong
Right now the preview does:

`<style>${liveCss}</style> ... ${liveHtml} ... <script>${liveJs}</script>`
If Claude puts <html> or <!DOCTYPE> tags inside the html field (which it often does), the browser double-parses it and breaks. Also if JS uses DOMContentLoaded but the script runs after elements exist — timing breaks.

THE FIX: Ask Claude for ONE field: full_html — the complete document start to finish. Then just do <iframe srcDoc={full_html}>. Done.
